# View

The `view` function is responsible for rendering your `Model` into a visual representation. In Artisanal, this is typically a `String` that will be printed to the terminal.

## The View Signature

```dart
String view()
```

## Advanced Views (Metadata)

While most views return a `String`, you can also return a `View` object to provide declarative metadata to the terminal. This allows you to control things like the window title, progress bars, and terminal colors without manually emitting ANSI sequences.

```dart file=../../../example/tui/metadata_demo.dart
```

### View Properties

- **`content`**: The actual string content to render.
- **`windowTitle`**: Sets the terminal window title.
- **`progressBar`**: Controls the terminal's native progress bar (OSC 9;4) using `TerminalProgressBar`.
- **`backgroundColor` / `foregroundColor`**: Overrides the terminal's default colors.
- **`cursor`**: Controls the cursor position and shape.

## Advanced Rendering (UV)

For more complex UIs, you can use the **Ultraviolet (UV)** renderer, which provides a cell-buffer based approach with layers and compositing. See the [Advanced Rendering](../advanced-rendering/overview) section for more details.
