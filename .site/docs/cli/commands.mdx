# CLI Command Reference

This page is a quick command index. For guides, start with:
- [CLI Overview](./overview)
- [Migrations](./migrations)
- [Seeding](./seeding)
- [Schema tooling](./schema)

## Global patterns

- The CLI reads `orm.yaml` (auto-discovered, or use `--config`).
- Use `--connection <name>` to select a connection block.
- Many commands support `--database` to override the driver database option.

## Commands

### `init`

Scaffolds `orm.yaml` plus migration/seeder registries:

```bash
dart run ormed_cli:orm init
```

### `make`

Generates migrations or seeders:

```bash
dart run ormed_cli:orm make --name create_users_table
dart run ormed_cli:orm make --name create_users_table --format sql
dart run ormed_cli:orm make --name UserSeeder --seeder
```

### `migrate`

Applies pending migrations:

```bash
dart run ormed_cli:orm migrate
dart run ormed_cli:orm migrate --limit 5
dart run ormed_cli:orm migrate --pretend
```

### `migrate:export`

Exports `up.sql` and `down.sql` for migrations:

```bash
dart run ormed_cli:orm migrate:export --out database/migration_sql
dart run ormed_cli:orm migrate:export --out database/migration_sql --all
```

### `migrate:status`

```bash
dart run ormed_cli:orm migrate:status
dart run ormed_cli:orm migrate:status --pending
```

### `migrate:rollback`

```bash
dart run ormed_cli:orm migrate:rollback
dart run ormed_cli:orm migrate:rollback --steps 3
```

### `migrate:fresh` / `migrate:reset` / `migrate:refresh`

```bash
dart run ormed_cli:orm migrate:fresh
dart run ormed_cli:orm migrate:reset
dart run ormed_cli:orm migrate:refresh
```

### `seed`

```bash
dart run ormed_cli:orm seed
dart run ormed_cli:orm seed --class UserSeeder
dart run ormed_cli:orm seed --pretend
```

### `schema:dump`

```bash
dart run ormed_cli:orm schema:dump
dart run ormed_cli:orm schema:dump --prune
```

### `schema:describe`

```bash
dart run ormed_cli:orm schema:describe
dart run ormed_cli:orm schema:describe --json
```

### `db:wipe`

```bash
dart run ormed_cli:orm db:wipe
dart run ormed_cli:orm db:wipe --drop-views
```
